## Build — first production build + workerd smoke test (ran during this audit)

- `opennextjs-cloudflare build`: **succeeds, zero warnings**; all routes prerender (○ static /
  ● SSG); Worker saved. The dev-only `require is not defined` quirk did NOT reproduce in prod
  (content-page.tsx is unimported/unbundled).
- `opennextjs-cloudflare preview` (real workerd runtime): **every ● SSG dynamic-segment route
  404s** — /team, /mission, all /methods/[slug], /specialties/[slug], /blog/[slug], every
  bespoke /[slug] page. ○ static routes (/, /careers, /contact, /contacto, /blog, /methods,
  /specialties), static assets, and `/_next/image` (IMAGES binding) all 200.
- **Root cause (confirmed)**: `open-next.config.ts` is `defineCloudflareConfig({})` — no
  incremental cache. OpenNext serves `generateStaticParams` prerenders THROUGH the incremental
  cache; with none configured the Worker has no source for them → 404. The config was copied from
  kevincameron, which has **zero** `generateStaticParams` routes (all ○ static), so the empty
  config was correct there and silently wrong here.
- **Fix**: use the shipped `static-assets-incremental-cache` override ("should only be used for
  applications that do NOT want revalidation and ONLY want to serve prerendered data" — exactly
  this site): in `open-next.config.ts`,
  `import staticAssetsIncrementalCache from "@opennextjs/cloudflare/overrides/incremental-cache/static-assets-incremental-cache"`
  and pass `{ incrementalCache: staticAssetsIncrementalCache }`. No new wrangler bindings needed
  (it reads prerenders bundled into assets under `cdn-cgi/_next_cache`). Rebuild + re-smoke.
- Deploy would have shipped a site where nearly every content page 404s — this is the #1 blocker.
